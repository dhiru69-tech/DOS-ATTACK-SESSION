01. INTRODUCTION

DoS (Denial of Service)

Single attacker floods server with requests.

DDoS (Distributed DoS)

Multiple attackers/botnets flood server simultaneously.

Business Impact

  • Operational Downtime
  • Financial Loss
  • Reputation Damage
ANALOGY: THE SHOP
BLOCKED

"One person blocking the entrance (DoS) vs. Thousands rushing in (DDoS)."

02. NETWORKING BASICS

IP Address

Digital address of a device (like a home address).

Server

System providing services/resources to others.

Bandwidth

Network capacity. Think: Width of the road.

Packets

Small data units transmitted over network.

Client-Server

Relationship between users (Client) and providers (Server).

04. ATTACK TYPES (Part 1)

Ping Flood

ICMP

Full Form: Internet Control Message Protocol

NORMAL USE Network diagnostics (Ping command).
MISUSE Excessive ICMP requests overload server.

"Analogy: Doorbell pressed thousands of times per second."

Defense: Rate limiting, Firewall rules.

SYN Flood

TCP

Full Form: Synchronize (TCP Handshake)

NORMAL USE Start TCP connection.
MISUSE Many SYN requests, No ACK sent back → Half-open connections.

"Analogy: Reserving seats in a restaurant but never showing up."

Defense: SYN Cookies.

04. ATTACK TYPES (Part 2)

HTTP Flood

Layer 7

Full Form: HyperText Transfer Protocol

NORMAL USE Browser-Server communication (GET/POST).
MISUSE Massive GET/POST requests overload server resources.

"Analogy: Thousands of fake online food orders."

Defense: WAF, CAPTCHA.

Amplification

DNS/NTP

Full Form: Domain Name System / Network Time Protocol

NORMAL USE Resolving domains, Syncing clocks.
MISUSE Small spoofed request → Huge response sent to victim.

"Analogy: Shouting in a cave → Echo comes back 100x louder."

Defense: Anti-spoofing filters.

06. DEFENSE TECHNIQUES (Part 1)

Firewall & IDS/IPS

Intrusion Detection/Prevention System.

Role: Block suspicious packets.

Analogy: Security guard at gate.

Rate Limiting

Restrict requests per user.

Role: Prevent overload from single IP.

Analogy: Limit shop entry frequency.

Load Balancing

Distribute traffic.

Role: Avoid single server overload.

Analogy: Multiple bank counters.

06. DEFENSE TECHNIQUES (Part 2)

CDN

Content Delivery Network

Role: Distribute content globally. Absorb traffic at the edge.

Analogy: Multiple warehouses serving customers.

Cloud DDoS Protection

Cloudflare, Akamai, AWS Shield

Role: Real-time filtering in the cloud. Scrub bad traffic.

Analogy: Private security army.

07. KEY TAKEAWAYS

  • DoS/DDoS is both a Technical and Business risk.
  • Prevention requires Layered Defenses (Firewall + CDN + Rate Limit).
  • Attackers exploit weaknesses, but Proactive Monitoring = Resilience.
